1. What is HTML?
Answer: HTML (HyperText Markup Language) is the standard markup language used to create webpages and structure content on the internet. It is not a programming language.

Defines elements like

  • Headings
  • Paragraphs
  • Images
  • Forms
  • Links
  • Tables

Example:

<h1>This is heading</h1>
<p>This is paragraph</p>

👀 Output in browser:

This is heading

This is paragraph

2. Difference between HTML and HTML5?
Answer: HTML5 is the latest version with semantic tags, native multimedia, and better browser capabilities.
HTML HTML5
Older version Latest version
Limited multimedia support Native audio/video support
No semantic tags Added semantic tags
Needed plugins like Flash No plugins needed
Less storage support localStorage/sessionStorage added

Example:

<header></header>
<nav></nav>
<section></section>
<article></article>
<footer></footer>

👀 Output in browser:

<header> — top section
<nav> — navigation
<section> — grouped content
<article> — self-contained content
<footer> — bottom section

These render just like divs visually — the difference is meaningful semantics for SEO and screen readers.

6. What is DOCTYPE in HTML?
Answer: DOCTYPE tells the browser which HTML version is used so pages render correctly.

Example:

<!DOCTYPE html>

👀 Output in browser:

⚠ No visible output on the page — DOCTYPE must be the very first line of an HTML file. It instructs the browser to use modern HTML5 rendering. Without it, the browser falls into "quirks mode" and may render the page incorrectly.

11. Difference between GET and POST?
Answer: GET sends data in URL, POST sends data in request body.
GET POST
Data visible in URL Data hidden
Less secure More secure
Used for fetching Used for submitting
Limited data size Large data allowed

Example:

<!-- GET: search query in URL -->
<form method="get" action="/search">
  <input name="q" placeholder="Search">
</form>

<!-- POST: form data in request body -->
<form method="post" action="/register">
  <input name="email" type="email">
</form>

👀 Output in browser:

GET: After submit, URL becomes: /search?q=your+text

POST: After submit, URL stays: /register — data is hidden in request body

15. What is localStorage and sessionStorage?
Answer: Both store data in browser. localStorage is persistent; sessionStorage is tab-session based.
localStorage sessionStorage
Permanent storage Temporary storage
Remains after closing browser Cleared after tab close
Larger storage Session based

Example:

localStorage.setItem("name", "Vishal");
sessionStorage.setItem("user", "admin");

👀 Output in browser:

⚠ No visible output on the page. Data is stored silently in browser memory. Open DevTools (F12) → Application → Local Storage / Session Storage to see stored values. Try: localStorage.setItem("name","Vishal") in the browser Console tab.

21. Difference between async and defer?
Answer: Both are used with external scripts, but loading and execution order differ.
async defer
Loads independently Loads in order
Executes immediately Executes after HTML parsing
May break order Maintains order

Example:

<script async src="app.js"></script>
<script defer src="app.js"></script>

👀 Output in browser:

async: Script downloads in parallel → runs immediately when ready (may run before HTML is fully parsed)
defer: Script downloads in parallel → waits until HTML fully parsed, then runs in order

No visible change on page. The difference is loading behavior. Use defer for most scripts to avoid blocking rendering.

26. What happens when webpage loads in browser?
Answer: Browser follows a rendering pipeline from request to final painted UI.
  1. Browser requests webpage
  2. Server sends HTML
  3. Browser parses HTML
  4. CSS applied
  5. JavaScript executed
  6. Page rendered

Example:

<!-- Browser parses HTML to DOM, then paints after CSSOM + layout -->

👀 Output in browser:

1. You type URL → Browser sends request to server
↓
2. Server sends back HTML file
↓
3. Browser builds DOM from HTML
↓
4. CSS loaded → CSSOM built
↓
5. JavaScript runs
↓
6. Render tree → Layout → Paint → Page visible

Interviewers usually check semantic understanding, practical coding, accessibility awareness, clean structure, and confidence while explaining.

Frontend interviews often combine HTML, CSS, JavaScript, responsive design, and browser behavior.

38. What are inline, internal, and external CSS?
Answer: CSS can be written inline, inside style tags, or in external files.

Example:

Inline:
<p style="color:red">Text</p>

Internal:
<style>
p {
  color: red;
}
</style>

External:
<link rel="stylesheet" href="style.css">

👀 Output in browser:

Inline: This text is red ← style applied directly on the element

Internal: This text is also red ← styled by a <style> block in <head>

External: This text is red too ← styled via an external .css file (linked in <head>)

All three produce the same visual result — the difference is where the CSS is written.

43. Difference between script in head and body?
Answer: Scripts in head can block rendering unless async/defer is used. End-of-body scripts usually allow faster initial render.

Example:

<script defer src="app.js"></script>

👀 Output in browser:

In <head> without defer: Script blocks HTML parsing → page renders slower
In <head> with defer: Script downloads in background → runs after HTML is fully parsed → faster render ✓
At end of <body>: Same effect as defer — HTML is parsed first, script runs last
44. What is accessibility in HTML?
Answer: Accessibility means making websites usable for disabled users, screen readers, and keyboard navigation.

Examples include semantic tags, alt text, and form labels.

Example:

<label for="name">Name</label>
<input id="name" type="text" aria-required="true">

<img src="profile.jpg" alt="Profile photo of user">

👀 Output in browser:

← clicking this label focuses the input below

Screen readers announce: "Name, text field, required." Without the label, they would just say "text field" — useless.

49. What is progressive enhancement?
Answer: Progressive enhancement means building basic functionality first, then adding advanced features.

Goal: better compatibility and accessibility.

Example:

<button type="submit">Save</button>
<!-- Works without JavaScript; JS can later add loading spinner and validation -->

👀 Output in browser:

Base layer — works without JavaScript:

This button works even with JavaScript disabled. Progressive enhancement means you build this foundation first, then layer on enhancements (spinner, validation, async save) for browsers that support them.

50. What are deprecated tags in HTML?
Answer: Deprecated tags are old tags that should no longer be used in modern HTML.

Examples: <font>, <center>, <marquee>

Use CSS instead.

Example:

<!-- Old: <font color="red">Text</font> -->
<p style="color: red;">Text</p>

👀 Output in browser:

Old deprecated way (<font>):

Text ← same red result

Modern CSS way (style="color:red"):

Text ← same red result, correct approach

Both look identical — but <font> is removed from HTML5. Always use CSS for presentation.
