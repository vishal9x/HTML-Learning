🟢 Beginner — Core HTML Fundamentals (Q1–Q15)
1. What is HTML? How is it different from HTML5?
Answer: HTML (HyperText Markup Language) is the standard language for building web pages. HTML5 is the latest version (2014) that added semantic elements, native multimedia, Canvas, Web Storage, Geolocation, and more — removing the need for plugins like Flash.
Feature HTML HTML5
Doctype Long and complex <!DOCTYPE html>
Multimedia Needs Flash plugin Native <audio>, <video>
Graphics No native support <canvas>, SVG
Forms Basic input types email, date, range, color…
Semantic tags Only <div> <header>, <article>…
Storage Cookies only localStorage, sessionStorage

👀 Output in browser:

<!DOCTYPE html>  ← HTML5 doctype (simple)
<html lang="en">
  <head>...</head>
  <body>...</body>
</html>
2. What is the basic structure of an HTML document?
Answer: Every HTML document has a DOCTYPE declaration, an <html> root, a <head> section for metadata, and a <body> section for visible content.

Example:

<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8">
    <title>My Page</title>
  </head>
  <body>
    <h1>Hello World</h1>
    <p>This is my page.</p>
  </body>
</html>

👀 Output in browser:

Hello World

This is my page.

The <head> content is invisible. Only what's inside <body> renders on the page.

3. What is the difference between a tag, element, and attribute?
Answer: A tag is the code inside angle brackets (<p>). An element is the full unit: opening tag + content + closing tag. An attribute adds extra information to a tag, written as name="value".

Example:

<a href="https://google.com" target="_blank">Visit Google</a>
 ↑ tag      ↑ attribute                        ↑ content      ↑ closing tag
 └──────────────── the full element ──────────────────────────┘

👀 Output in browser:

Visit Google

The link text "Visit Google" is visible. href and target are attributes — they're invisible but control behaviour.

Answer: Void elements have no closing tag and no content — they are complete on their own. Common examples: <br>, <img>, <hr>, <input>, <meta>, <link>.

<input type="text">               ← form input

5. What is DOCTYPE and why is it needed?
Answer: <!DOCTYPE html> tells the browser this is an HTML5 document. Without it, the browser enters "quirks mode" and renders the page using outdated rules, causing layout inconsistencies.

Example:

<!DOCTYPE html>   ← HTML5 (modern)
<html lang="en">

👀 Output in browser:

⚠ No visible output — DOCTYPE is an instruction to the browser, not content. It must be the very first line. With it: standards mode. Without it: quirks mode (unpredictable rendering).
6. What is the difference between block-level and inline elements?
Answer: Block elements start on a new line and stretch full width (<div>, <p>, <h1>). Inline elements stay in the flow of text and only take as much width as their content (<span>, <a>, <strong>).

Example:

<div>I am block — full row</div>
<div>Second block — new row</div>
<span>Inline</span> stays on same line as <span>this</span>

👀 Output in browser:

I am block — takes full row
Second block — starts on new row

Inline stays on same line as this

8. What is the difference between id and class attributes?
Answer: id is unique — only one element per page can use a given id. class can be reused across many elements. CSS uses #id and .class respectively.

Example:

<div class="card">Card 1</div>
<div class="card">Card 2</div>

Answer: Semantic elements clearly describe their purpose: <header>, <nav>, <main>, <article>, <section>, <aside>, <footer>. They improve: (1) SEO — search engines understand content structure, (2) Accessibility — screen readers announce landmarks, (3) Code readability.

11. How do you create a hyperlink? What are absolute vs relative URLs?
Answer: Use the <a href="..."> tag. An absolute URL includes the full address (protocol + domain). A relative URL is relative to the current page's location.

Example:

<!-- Absolute URL -->
<a href="https://google.com">Google</a>

<!-- Relative URL -->
<a href="/about.html">About Us</a>

<!-- Same page anchor -->
<a href="#section1">Jump to Section</a>

<!-- Email link -->
<a href="mailto:user@example.com">Email Us</a>

👀 Output in browser:

Google ← absolute URL, always works anywhere

About Us ← relative URL, resolves from this site's root

Email Us ← opens the user's email client

12. What is the alt attribute in <img> and why is it important?
Answer: The alt attribute provides a text description of an image. It matters for: (1) Accessibility — screen readers read it aloud, (2) SEO — search engines index it, (3) Broken image fallback — displayed when the image fails to load.

Example:

<img src="dog.jpg" alt="Golden retriever playing in a park">

14. How do you make a link open in a new tab safely?
Answer: Use target="_blank" to open in a new tab. You must also add rel="noopener noreferrer" for security — without it, the new tab can access and manipulate your page via window.opener.

Example:

<!-- ❌ Insecure -->
<a href="https://example.com" target="_blank">Link</a>

<!-- ✅ Secure -->
<a href="https://example.com" target="_blank" rel="noopener noreferrer">Link</a>

👀 Output in browser:

Open in new tab (secure)

noopener — prevents the new tab from accessing window.opener (stops phishing attacks). noreferrer — hides the referring URL from the destination site.

15. What is the <title> tag and what are meta tags?

18. What is HTML5 form validation? What attributes are used?
Answer: HTML5 has built-in client-side validation via attributes: required, type (e.g., email, url), min/max, minlength/maxlength, and pattern. The browser validates on submit without any JavaScript.

Example:

<input type="email" required placeholder="Email">
<input type="text"  minlength="3" maxlength="20" required>
<input type="number" min="18" max="99">
<input type="text"  pattern="[0-9]{10}" placeholder="Phone (10 digits)">

👀 Output in browser: (try submitting with bad data)

Click Submit with an empty field or a bad email — the browser shows built-in error messages with zero JavaScript.

<input type="email" id="email" name="email">

26. What is Canvas in HTML5?
Answer: <canvas> provides a pixel-based drawing surface controlled entirely by JavaScript. Used for games, charts, and animations. Raster-based — it does not scale cleanly when zoomed.

Example:

<canvas id="demo" width="300" height="100"></canvas>
<script>
  const ctx = document.getElementById('demo').getContext('2d');
  ctx.fillStyle = '#2563eb';
  ctx.fillRect(10, 10, 200, 60);
  ctx.fillStyle = 'white';
  ctx.font = '20px Arial';
  ctx.fillText('Canvas Demo', 60, 48);
</script>

👀 Output in browser:

The blue rectangle and white text above are drawn pixel-by-pixel by JavaScript using the Canvas 2D API.

27. What is SVG? How does it differ from Canvas?

DOM access Yes — each shape is a DOM node No — just pixels
Best for Icons, logos, charts Games, complex animations

29. What is an iframe? What is the sandbox attribute?
Answer: <iframe> embeds another webpage inside the current page. The sandbox attribute restricts what the embedded content can do (run scripts, submit forms, navigate parent). Always use sandbox when embedding untrusted content.

Example:

<!-- Unsafe — no restrictions -->
<iframe src="https://example.com"></iframe>

<!-- Safe — sandbox restricts everything, re-enable only what's needed -->
<iframe src="https://example.com"
        sandbox="allow-scripts allow-same-origin"
        title="Embedded page"></iframe>

👀 Output in browser:

The above box shows example.com rendered inside this page via <iframe>.

Example:

document.querySelector('h1').style.color = 'red';

32. What is the difference between async and defer on <script> tags?
Answer: Both allow the script to download without blocking HTML parsing. async runs the script immediately when downloaded (may interrupt parsing, order not guaranteed). defer waits until HTML is fully parsed, then runs scripts in order. Use defer for most app scripts.

Example:

<script async src="analytics.js"></script>  ← runs when ready, any order
<script defer src="app.js"></script>        ← runs after DOM ready, in order

👀 Output in browser:

async: Downloads in parallel → runs immediately when ready → may interrupt HTML parsing
defer: Downloads in parallel → waits for full HTML parse → runs in document order ✓
Neither: Blocks HTML parsing until script downloads and runs ❌
33. What are data-* attributes?
Answer: Custom data-* attributes let you store extra information on DOM elements without polluting the markup. JavaScript reads them via element.dataset. They're invisible on the page and perfect for event delegation patterns.

Example:

// JavaScript:
const btn = document.querySelector('button');
console.log(btn.dataset.action); // "delete"
console.log(btn.dataset.id);     // "42"

👀 Output in browser: (click to read data attributes)

This element has hidden: data-id="101", data-role="admin", data-user="Vishal"

🔴 Advanced — Performance, Security, Accessibility & Modern APIs (Q41–Q55)
41. What does rel="noopener noreferrer" do and why does it matter for security?
Answer: When using target="_blank", the new tab can access your page via window.opener, enabling phishing attacks (redirecting your original tab). rel="noopener" nullifies window.opener. rel="noreferrer" additionally hides the Referer header. Always use both.

Example:

<!-- ❌ Vulnerable: new tab can redirect your page -->
<a href="https://evil.com" target="_blank">Link</a>

<!-- ✅ Safe: window.opener is null -->
<a href="https://example.com" target="_blank" rel="noopener noreferrer">Link</a>

👀 Output in browser:

Without noopener: The new tab can run window.opener.location = "https://phishing.com" and silently redirect your original tab to a fake login page.

With noopener: window.opener is null in the new tab — the attack is blocked.

42. What is lazy loading for images and when should you avoid it?
Answer: loading="lazy" defers image loading until the image is near the viewport, reducing initial page load weight. Avoid it for: (1) Hero images or any above-the-fold content — it delays what the user sees first, (2) Your LCP (Largest Contentful Paint) element — it hurts Core Web Vitals.

Example:

<!-- ✅ Lazy load offscreen images -->
<img src="thumbnail.jpg" alt="Product" loading="lazy">

<!-- ❌ Don't lazy load hero/above-fold -->
<img src="hero.jpg" alt="Hero" loading="lazy">

<!-- ✅ Use fetchpriority for critical images -->
<img src="hero.jpg" alt="Hero" fetchpriority="high">

👀 Output in browser:

✅ loading="lazy" — thumbnail at bottom of page — loads when scrolled near
❌ loading="lazy" on hero image — delays what user sees immediately
43. What is ARIA? When should you NOT use ARIA?
Answer: ARIA (Accessible Rich Internet Applications) adds role, aria-label, aria-expanded etc. to help screen readers understand custom UI components. The #1 rule: no ARIA is better than bad ARIA. Don't use ARIA when a native HTML element already does the job.

Example:

<!-- ❌ Recreating a button with ARIA -->
<div role="button" onclick="submit()">Submit</div>

<!-- ✅ Use the native element — no ARIA needed -->
<button onclick="submit()">Submit</button>

<!-- ✅ ARIA is needed for custom components -->
<div role="dialog" aria-labelledby="title" aria-modal="true">
  <h2 id="title">Confirm Action</h2>
</div>

👀 Output in browser:

← aria-label="Close dialog" tells screen readers what this icon button does

44. What is the <dialog> element and how do showModal() / close() work?

45. What is the <template> tag?
Answer: <template> holds HTML that is parsed but not rendered. It's inert — scripts don't run, images don't load. JavaScript clones it and inserts it into the DOM when needed. Perfect for reusable components and dynamic content.

Example:

<template id="card-template">
  <div class="card">
    <h3>Card Title</h3>
    <p>Card content here</p>
  </div>
</template>

<script>
  const tpl = document.getElementById('card-template');
  const clone = tpl.content.cloneNode(true);
  document.body.appendChild(clone);
</script>

👀 Output in browser: (click Stamp Template)

Each click stamps a new card — click multiple times.

46. What are Web Components?
Answer: Web Components are a set of browser APIs to create reusable custom HTML elements. Three core technologies: (1) Custom Elements — define your own tags, (2) Shadow DOM — encapsulated styles and markup, (3) HTML Templates — reusable markup via <template>.

Example:

<my-card title="Hello"></my-card>

<script>
class MyCard extends HTMLElement {
  connectedCallback() {
    this.innerHTML = `<div class="card">${this.getAttribute('title')}</div>`;
  }
}
customElements.define('my-card', MyCard);
</script>

👀 Output in browser:

⚠ Web Components require JavaScript and run in a live browser. The custom element renders like any other HTML element once defined. Shadow DOM keeps the component's styles isolated from the rest of the page.
47. What is the tabindex attribute?
Answer: tabindex controls keyboard focus order. 0 = focusable in natural DOM order. -1 = focusable via JS only, skipped by Tab key. Positive numbers set custom order (avoid in practice — it breaks natural flow).

Example:

<button tabindex="0">Normal focus order</button>
<div tabindex="0">This div is now keyboard focusable</div>

48. What is the contenteditable attribute?

49. What are preload, prefetch, and preconnect? When to use each?
Answer: All three are resource hints in <head>. preload — fetch this resource NOW, it's needed for this page (hero image, critical font). prefetch — low-priority fetch for resources the next navigation will need. preconnect — warm up DNS + TCP + TLS to a domain before the resource is requested.

Example:

<!-- Warm up Google Fonts connection -->
<link rel="preconnect" href="https://fonts.googleapis.com">

<!-- Fetch hero image immediately -->
<link rel="preload" as="image" href="hero.jpg">

<!-- Pre-fetch next page's bundle -->
<link rel="prefetch" href="/about.js">

👀 Output in browser:

preconnect → Warms up connection early. No resource fetched yet.
preload → Starts fetching hero.jpg immediately at high priority.
prefetch → Downloads about.js quietly in idle time for the next page.
50. What is Subresource Integrity (SRI)?
Answer: SRI lets you verify that an externally loaded file (from a CDN) hasn't been tampered with. You add an integrity attribute containing the expected file hash. If the hash doesn't match the downloaded file, the browser blocks it entirely.

Example:

<script
  src="https://cdn.example.com/library.js"
  integrity="sha384-abc123xyz..."
  crossorigin="anonymous">
</script>

👀 Output in browser:

1. Browser downloads library.js from the CDN

2. Browser hashes the downloaded file content

3a. Hash matches → script runs normally ✅

3b. Hash mismatch → browser blocks the script entirely ❌ (you're protected from supply chain attacks)

52. What are Web Workers in HTML5?
Answer: Web Workers run JavaScript in a background thread — separate from the main UI thread. This prevents heavy computations from freezing the page. Workers cannot directly access the DOM; they communicate with the main thread via postMessage().

Example:

// main.js
const worker = new Worker('worker.js');
worker.postMessage({ data: largeArray });
worker.onmessage = e => console.log('Result:', e.data);

// worker.js
self.onmessage = e => {
  const result = heavyComputation(e.data);
  self.postMessage(result);
};

👀 Output in browser:

⚠ Web Workers run silently in the background — no visible output. The benefit is that the UI (scrolling, animations, clicks) stays responsive even while the worker handles an expensive operation like parsing a large file or number-crunching.
53. What is the lang attribute and what does it affect?
Answer: The lang attribute on <html> declares the document's language. It affects: (1) Screen reader pronunciation engines, (2) Browser hyphenation rules, (3) Google Translate source language detection, (4) CSS :lang() selector. Always set it, and override on specific elements for multilingual content.

Example:

<html lang="en">          ← whole page is English
  <blockquote lang="fr">  ← this quote is French
    Bonjour le monde
  </blockquote>
</html>

👀 Output in browser:

English sentence. «Bonjour le monde» — the French text has lang="fr" so screen readers switch pronunciation engine for that phrase.

54. What is Cumulative Layout Shift (CLS) and how do HTML choices reduce it?
Answer: CLS is a Core Web Vital measuring unexpected visual shifts during page load (e.g., an image popping in and pushing text down). Good score is below 0.1. Fix by: (1) always specifying width/height on images, (2) reserving space for ads, (3) using aspect-ratio in CSS.

Example:

<!-- ❌ No dimensions — causes layout shift -->
<img src="banner.jpg" alt="Banner">

<!-- ✅ Dimensions reserved — no shift -->
<img src="banner.jpg" alt="Banner" width="1200" height="400">

<!-- ✅ CSS aspect-ratio approach -->
<style>
  img { width: 100%; aspect-ratio: 3 / 1; }
</style>

👀 Output in browser:

❌ No width/height: image loads → text below jumps down → CLS score increases
✅ width/height set: browser reserves exact space → text stays in place → CLS = 0
55. How do you safely update the DOM to avoid XSS? (textContent vs innerHTML)
Answer: innerHTML parses the string as HTML — if user input contains <script> tags, they execute. This is an XSS vulnerability. textContent treats everything as plain text, never executes it. Always use textContent for user-provided content. Only use innerHTML when needed, and sanitize first with DOMPurify.

Example:

const input = '<img src="x" onerror="alert(\'hacked\')">';

// ❌ Dangerous — executes the onerror event
element.innerHTML = input;

// ✅ Safe — renders as literal text
element.textContent = input;

// ✅ Safe when HTML is truly needed
element.innerHTML = DOMPurify.sanitize(input);

👀 Output in browser: (try both)

textContent (safe) renders this as literal text:

The <img onerror=alert(1)> is shown as text — not executed. With innerHTML it would trigger an alert popup (XSS).
